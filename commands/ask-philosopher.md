# Onto Philosopher (legacy compatibility alias)

> **⚠ NON-CANONICAL (legacy command surface)**: This command is preserved for non-canonical use only. The canonical review path does not use 'philosopher'.

Legacy alias for the axiological alignment perspective.

**Scope narrowing**: This alias preserves only the value-alignment questioning behavior. It does NOT preserve the former Philosopher's coordinator or synthesis meaning, which has been decomposed into the canonical 9-lens + synthesize pipeline (see `processes/review/` contracts).

Use `/onto:ask-axiology` for new usage.
This compatibility command continues to route the question to the axiological lens so existing workflows do not break during productization.

Agent ID: `axiology` (formerly `onto_axiology`)

**Domain selection**: Append `@{domain}` to specify a domain, or `@-` for no-domain mode. If omitted, the Domain Selection Flow runs interactively.

Read the project-local copies of `process.md` (common definitions), `learning-rules.md` (learning storage rules), and `processes/question.md` — these are located at the repository root and repository-relative paths respectively. When executed from a plugin install, the runtime resolves these paths from the plugin's install directory; no hard-coded absolute path is required.

- **consumers**: M-05 (dep graph 축·활동 bucket 참조), M-06 (work item activity/axis 필드 값 resolve), M-07 (lifecycle balance 축·활동 교차 집계), **M-08 refresh protocol (G-6 추가, v1.1)** — Stage 3 entrypoint align 모니터 + lexicon_version diff 추적, Expansion Protocol (활동·축 추가·삭제 trigger), consolidated v3.x (frontmatter `activity_enum_ref` / `axis_enum_ref` resolve 대상).

## DR-M04-02: activity_enum_ref / axis_enum_ref frontmatter 필드 의무화 (DL-014 해소)

- **status**: applied
- **origin_ref**: DL-014 (v2-review evolution F2 Recommend — enum 진화 hook)
- **context**: DR-M04-01 에서 authority/core-lexicon.yaml 의 canonical seat 을 확정한 후, 이 seat 을 참조하는 frontmatter 필드 규약을 M-06 work item canonical output + consolidated 재revision 에 의무화한다. 기존 consolidated v3.2 의 `activity_enum_ref: "...20260413-onto-direction.md#§1.2"` 형식을 canonical seat 참조 형식으로 일괄 교체 (M-06 시점).
- **Decision**: work item schema 와 consolidated frontmatter 양쪽에 아래 두 필드 의무화.
- **default_rule**:
  - **필드 이름**: `activity_enum_ref`, `axis_enum_ref`.
  - **값 형식**: `authority/core-lexicon.yaml#<term_id>.v<lexicon_version>` (예: `authority/core-lexicon.yaml#activity_enum.v0.6.1`).
  - **참조 해석**: lexicon_version snapshot 에 bound. lexicon_version bump 시 downstream reader 는 diff 확인 후 재파싱 (E-7 migration logic 연계, DL-027 에서 수렴).
  - **legacy 호환 기간 상한** (C-09 해소, v1.1): consolidated v3.x 기존 `#§1.2` / `#§1.5` 형식은 **M-06 재revision 시점 (latest)** 까지 일괄 교체. 그 전까지 두 형식 병존 허용 (reader 는 둘 다 resolve 가능). **상한 초과 시 Principal escalate** — M-06 종료까지 미교체 시 schema validator 가 새 work item reject. M-06 onto-todo.md 의 신규 work item 은 처음부터 신규 형식만 사용.
  - **halt 실행 주체** (C-09 해소, v1.1): "halt + Principal 질의" 의 실행 주체는 **활성 stage agent (현재 진행 중인 meta task 의 수행 주체)** — M-05 dep graph 작성 agent / M-06 work item 작성 agent / M-08 refresh agent 등. 구체 agent 가 없는 standalone reader 는 halt 후 stderr 로 보고 + 종료.
  - **version bump 규약**: activity_enum / axis_enum 값 추가·삭제·rename 시 lexicon_version minor bump (additive). 의미 변경 시 major bump. Expansion Protocol Migration Contract (v5.1 meta) 연계.
- **escalation_trigger**:
  - (1) lexicon_version 과 ref 에 적힌 version 불일치 → reader 가 halt + Principal 질의 (활성 stage agent 가 실행).
  - (2) 필드 자체 부재 (schema 위반) → M-06 canonical output 에서 reject.
  - (3) **anchor unresolvable** (v1.1 추가, C-01 재발 방지): `<ref>#<anchor>` 형식의 enum_ref 가 대상 파일에 실제 anchor (heading slug 또는 explicit `<a id>`) 를 갖지 않으면 schema validator 가 **reject** + ledger 에 신규 DL 등재. 본 trigger 는 모든 enum_ref (activity_enum_ref, axis_enum_ref, canonicality_enum_ref, disposition_enum_ref, lifecycle_status_enum_ref) 에 적용.
  - (4) 여러 consumers 가 서로 다른 version 을 pin 한 상태에서 migration 필요 → M-08 refresh protocol 내 migration window.
- **owning_artifact**: 본 파일 (DR 선언) + `authority/core-lexicon.yaml` (실제 term seat) + `development-records/design/20260413-onto-todo.md` (work item schema 반영).
- **consumers**: M-05 / M-06 / M-07 / M-08 / Expansion Protocol. consolidated 재revision. 미래 work item 생성·migration 도구.
- **applied_cases**:
  - 본 M-04 Phase A 실행 중 `20260413-onto-todo.md` 초판 frontmatter 에 두 필드 반영. DL-014 resolved. (v1)
  - **v1.1 (C-01 즉시 적용)**: `lifecycle_status_enum_ref` 의 broken anchor (`#M-04-field-schema`) 를 self-ref (`#1-2-enum-값-목록-본-파일-시점-snapshot`) 로 교체. 본 DR 의 escalation_trigger(3) 이 본 케이스에서 reject 했어야 하나 v1 에서 미적용 — v1.1 trigger 추가로 향후 동종 결함 사전 차단.

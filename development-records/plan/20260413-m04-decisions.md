## DR-M04-01: activity field canonical seat 확정 (DL-013 해소)

- **status**: applied
- **origin_ref**: DL-013 (v5.1 meta M-04 Phase A scope)
- **context**: v5.1 meta 에서 M-06/M-07/Expansion 이 `activity` field 를 canonical authority 로 읽도록 문구 정렬 이 M-04 "실행 중 자연 반영" 항목으로 지정됨. consolidated v3.2 까지는 `activity_enum_ref: development-records/design/20260413-onto-direction.md#§1.2` 로 §1 정본 직접 참조를 임시 유지했다. M-04 Phase A 에서 `authority/core-lexicon.yaml` (rank 1 authority SSOT) 의 canonical seat 으로 이관한다.
- **Decision**: `authority/core-lexicon.yaml` 에 §1.activity + §1.axis canonical seat 추가. `activity_enum` term + `axis_enum` term 형식. 의미는 §1 정본(2026-04-13) 과 동기화만 — 개정 금지. 기존 `entrypoint` entity (Stage 2 scope) 는 건드리지 않음 (Stage 3 scope 에서 Principal 판단).
- **default_rule**:
  - **활동(activity) 값**: `review`, `design`, `reconstruct`, `learn`, `govern` (§1.2 5 활동). legacy_aliases: `build → reconstruct` (DR-M00-04 매핑), `ask → retired` (§1.2 폐기 선언, 대체 없음).
  - **축(axis) 값**: `A`, `B`, `C`, `D` (§1.5 4 축). 각 값의 성격:
    - A = entrypoint 구현 (활동별)
    - B = 기반 인프라 (공통)
    - C = 자율성 진화 (메타)
    - D = 상시 원칙 (교차, lexicon provisional lifecycle)
  - 두 seat 은 `activity_enum` / `axis_enum` 별개 term 으로 등재. 둘 사이 관계는 §1.5 "축 간 의존" 에 의해 derived (A 는 5 활동 entrypoint 구현 소재지).
- **escalation_trigger**: (1) §1 정본 §1.2 또는 §1.5 개정 시 본 DR 재실행 + legacy_aliases 갱신. (2) 새 활동·축 도입 제안 시 Expansion Protocol 경유 (Principal 직접 승인 only). (3) `authority/core-lexicon.yaml` 의 기존 entrypoint entity 와 activity_enum 사이 관계 정식화 요구가 발생하면 Stage 3 로 escalate.
- **owning_artifact**: `authority/core-lexicon.yaml` (rank 1 authority). 본 DR 은 ledger-level 참조.
- **consumers**: M-05 (dep graph 축·활동 bucket 참조), M-06 (work item activity/axis 필드 값 resolve), M-07 (lifecycle balance 축·활동 교차 집계), Expansion Protocol (활동·축 추가·삭제 trigger), consolidated v3.x (frontmatter `activity_enum_ref` / `axis_enum_ref` resolve 대상).
- **applied_cases**: 본 M-04 Phase A 실행 중 `authority/core-lexicon.yaml` 에 `activity_enum` + `axis_enum` term 추가. DL-013 resolved.

## DR-M04-02: activity_enum_ref / axis_enum_ref frontmatter 필드 의무화 (DL-014 해소)

- **status**: applied
- **origin_ref**: DL-014 (v2-review evolution F2 Recommend — enum 진화 hook)
- **context**: DR-M04-01 에서 authority/core-lexicon.yaml 의 canonical seat 을 확정한 후, 이 seat 을 참조하는 frontmatter 필드 규약을 M-06 work item canonical output + consolidated 재revision 에 의무화한다. 기존 consolidated v3.2 의 `activity_enum_ref: "...20260413-onto-direction.md#§1.2"` 형식을 canonical seat 참조 형식으로 일괄 교체 (M-06 시점).
- **Decision**: work item schema 와 consolidated frontmatter 양쪽에 아래 두 필드 의무화.
- **default_rule**:
  - **필드 이름**: `activity_enum_ref`, `axis_enum_ref`.
  - **값 형식**: `authority/core-lexicon.yaml#<term_id>.v<lexicon_version>` (예: `authority/core-lexicon.yaml#activity_enum.v0.6.0`).
  - **참조 해석**: lexicon_version snapshot 에 bound. lexicon_version bump 시 downstream reader 는 diff 확인 후 재파싱 (E-7 migration logic 연계, DL-027 에서 수렴).
  - **legacy 호환 기간**: consolidated v3.x 기존 `#§1.2` / `#§1.5` 형식은 v3.3 재revision (M-06 또는 M-08) 시점에 일괄 교체. 그 전까지 두 형식 병존 허용 (reader 는 둘 다 resolve 가능해야 함). M-06 onto-todo.md 초판부터는 신규 형식만 사용.
  - **version bump 규약**: activity_enum / axis_enum 값 추가·삭제·rename 시 lexicon_version minor bump (additive). 의미 변경 시 major bump. Expansion Protocol Migration Contract (v5.1 meta) 연계.
- **escalation_trigger**: (1) lexicon_version 과 ref 에 적힌 version 불일치 → reader 가 halt + Principal 질의. (2) 필드 자체 부재 (schema 위반) → M-06 canonical output 에서 reject. (3) 여러 consumers 가 서로 다른 version 을 pin 한 상태에서 migration 필요 → M-08 refresh protocol 내 migration window.
- **owning_artifact**: 본 파일 (DR 선언) + `authority/core-lexicon.yaml` (실제 term seat) + `development-records/design/20260413-onto-todo.md` (work item schema 반영).
- **consumers**: M-05 / M-06 / M-07 / M-08 / Expansion Protocol. consolidated 재revision. 미래 work item 생성·migration 도구.
- **applied_cases**: 본 M-04 Phase A 실행 중 `20260413-onto-todo.md` 초판 frontmatter 에 두 필드 반영. DL-014 resolved.

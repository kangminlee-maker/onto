- **status**: applied
- **origin_ref**: v5.1 meta M-05 spec "축 순서 재정립" + §1 정본 §1.5 축 간 의존
- **context**: §1 정본 §1.5 은 축 간 의존을 "A→B / C→{A,B} / D→{A,B,C}" 로 선언했으나, 이는 **선행 관계 선언** 이며 시간 축 실행 순서는 아직 풀어지지 않았다. v5.1 meta 는 이를 "D0 bootstrap → B → A → C + D continuing (cross-cutting lane)" 으로 시간 축으로 풀었다. 본 DR 은 이 풀림을 M-05 dep graph 의 **축 backbone** 으로 applied 확정한다.
- **Decision**: 축 실행 순서는 `D0 → B → A → C + D continuing`. 축 D 는 **완료되는 선행이 아닌 cross-cutting lane** 임을 명시.
- **default_rule**:
  - **D0 (bootstrap)**: 축 D 의 최소 성문화. BL-123 중 (a) lexicon provisional lifecycle 성문화 (seed → candidate → provisional → promoted 전이 규칙 + `lifecycle_status` 관리 위치), (b) 인용 금지 검증 도구. **완료 전 B/A/C 진입 금지**.

  - **A (entrypoint 구현)**: B 완료 후. 5 활동 (review / design / reconstruct / learn / govern) command·process·calling convention.
  - **C (자율성 진화)**: A·B 최소 1사이클 운영 후 설계. drift 판정 engine + 큐·승인 + 수준 0→1→2.

- **escalation_trigger**:

  - (2) D0 범위 확장 필요 (예: BL-123 외 다른 lexicon 관련 work 가 D0 에 포함) 시 DR-M05-NN 추가 또는 본 DR revise.

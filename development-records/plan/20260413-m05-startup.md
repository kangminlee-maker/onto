## 2. M-05 정의 (v5.1 meta 기준)

### 목적

축 간·활동 간 의존 그래프 모델링 — M-06 work item 작성의 입력 계약. Pre-draft 단계: task-level 의존은 M-07 책임.

### Scope

| 다루는 것 | 다루지 않는 것 |
|---|---|
| 축 (A/B/C/D) 간 의존 | task-level (W-X-NN) 의존 — M-07 |
| 활동 (review/design/reconstruct/learn/govern) 간 의존 | 구체 W-ID 할당 — M-06 |
| D bootstrap vs continuing 구분 | work item schema 변경 — M-04 reopen |
| Phase B sanity check 결정 (DL-031) | M-06 전수 work item 작성 |
| Compound/Phase sequencing 구조 (DL-015) | — |

### 축 순서 재정립 (v5.1 meta)

```
D0 bootstrap → B → A → C, D continuing
```

- **D0 (D bootstrap)**: 축 D 의 lexicon provisional lifecycle 의 최소 성문화 — 다른 축 시작 전 1회 완료
- **B**: 기반 인프라 (scope-runtime, readers, middleware). A0 = scope-runtime framework 추출 포함
- **A**: 활동별 entrypoint 구현 (B 위에서 작동)
- **C**: 자율성 진화 (A·B 최소 1사이클 운영 후 설계 가능)
- **D continuing**: D0 이후 cross-cutting lane — 모든 축에서 새 lexicon term 등장 시 등록·관리

이 순서는 §1.5 의 "A→B" / "C→{A,B}" / "D→{A,B,C}" 의존을 시간 축으로 풀어낸 것.

### 산출

1. **축·활동 의존 그래프** (text 또는 mermaid)
2. **D bootstrap vs ongoing 구분** 명문화
3. **DL-031 처리 결정** (Phase B sanity check)
4. **DL-015 처리** (Compound/Phase sequencing 구조)

- D0 이후 새 lexicon term 등장 시 등록·관리

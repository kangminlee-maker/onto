# Business Domain — 확장 시나리오

onto_evolution 에이전트가 각 시나리오를 시뮬레이션하여 기존 사업 체계의 깨짐 여부를 검증한다.

## Case 1: 신규 수익원 추가

- 기존 사업 체계에 새로운 수익원(제품/서비스)을 추가
- 검증: 기존 매출 인식 규칙이 새 수익원을 수용할 수 있는가? 키 구조에 충돌이 없는가?
- 검증: 새 수익원에 대한 K-IFRS 매핑이 완료되었는가?
- 검증: 기존 비용 구조에서 새 수익원의 원가를 귀속할 수 있는가?
- 영향 파일: structure_spec.md (매출 인식 키 구조), logic_rules.md (수익-비용 대응)

## Case 2: M&A (인수합병)

- 피인수 기업의 사업 체계를 기존 체계에 통합
- 검증: 매출 인식 규칙의 키 구조가 피인수 기업의 체계를 수용하는가?
- 검증: 회계 기준(K-IFRS) 적용 범위가 통합 후에도 일관되는가?
- 검증: 조직 구조 통합 시 의사결정 권한과 보고 체계의 충돌이 해소되는가?
- 검증: 과도기 병행 운영 기간과 완전 통합 일정이 정의되어 있는가?
- 영향 파일: dependency_rules.md (M&A 의존), structure_spec.md (키 구조)

## Case 3: 자동화 도입/확대

- 기존 수작업 프로세스를 자동화로 전환하거나, 자동화 범위를 확대
- 검증: 자동화 범위의 시작점과 끝점이 명확히 정의되어 있는가?
- 검증: 자동화 전 기준선(baseline) 측정이 수행되었는가?
- 검증: 수작업 결과와 자동화 결과의 불일치 판정 권한이 지정되어 있는가?
- 검증: "시간 절감"이 "비용 절감"으로 전환되는 경로가 설명되어 있는가?
- 영향 파일: logic_rules.md (자동화 범위 논리), competency_qs.md (Q11~Q13)

## Case 4: 시장/지역 확장

- 기존 사업을 새 시장 또는 새 지역으로 확장
- 검증: 매출 인식 규칙이 현지 회계 기준과 호환되는가?
- 검증: 가치 제안이 새 시장의 고객 세그먼트에 적합한가?
- 검증: 규제 환경 차이에 대한 대응이 정의되어 있는가?
- 검증: 공급망/운영 체계가 지역 확장을 수용하는가?
- 영향 파일: structure_spec.md (도메인 경계), dependency_rules.md (재무-운영 의존)

## Case 5: 비즈니스 모델 전환

- 제품 판매 → 구독 모델, 직접 판매 → 플랫폼 모델 등 근본적 수익 구조 변경
- 검증: 매출 인식 방식의 전환(일시 인식 → 기간 배분, 총액법 → 순액법)이 K-IFRS에 정합하는가?
- 검증: 기존 KPI(매출액, 이익률)가 새 모델에 적합한가? (MRR, LTV/CAC 등 신규 지표 필요 여부)
- 검증: 기존 조직 구조와 역량이 새 모델을 지원하는가?
- 검증: 전환 과도기의 매출 감소(J-커브)에 대한 재무 대응이 준비되어 있는가?
- 영향 파일: logic_rules.md (수익-비용 대응), structure_spec.md (수익 구조)

## Case 6: 기술 도입 (디지털 전환)

- AI, 클라우드, 데이터 분석 등 신기술의 사업 적용
- 검증: "첫 성공→확산" 전략의 진입점이 정의되어 있는가?
- 검증: 협조 확보와 변화관리의 순환 의존이 해소되어 있는가?
- 검증: 기술 도입 효과가 도입 시점이 아닌 통합 이후 시점에서 측정되는가?
- 검증: 기존 프로세스와의 통합 계획이 기술 도입 계획과 분리/연결되어 있는가?
- 영향 파일: dependency_rules.md (변화관리 순환 의존), logic_rules.md (변화관리 논리)

## Case 7: 조직 규모 확장

- 인원 증가, 부서 신설, 계층 추가 등 조직 규모 변화
- 검증: 의사결정 권한 체계가 확장된 조직을 수용하는가?
- 검증: 기존 프로세스의 처리 용량이 조직 규모와 정합하는가?
- 검증: 정보 전달 경로(보고 체계)가 규모 확장 후에도 지연 없이 동작하는가?
- 검증: 전략/전술/운영 계층 구분이 확장된 조직에서도 유효한가?
- 영향 파일: structure_spec.md (권한과 책임 분리), dependency_rules.md (조직-프로세스 의존)

## 관련 문서
- structure_spec.md — 사업 체계 구조, 매출 인식 키 구조
- dependency_rules.md — 전략-실행, 재무-운영, M&A 의존 규칙
- logic_rules.md — 수익-비용, ROI, 자동화, 변화관리 논리

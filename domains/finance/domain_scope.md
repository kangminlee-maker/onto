# Finance Domain — 도메인 범위 정의

## 분류 축

| 축 | 값 | 설명 |
|---|---|---|
| 보고 체계 | IFRS / K-GAAP | 적용 회계 기준 |
| 보고 단위 | 연결 / 별도 | 재무제표 작성 단위 |
| 보고 주기 | 연간 / 반기 / 분기 | 공시 빈도 |
| 원천 형식 | XBRL / HTML / PDF | 데이터 수집 원천의 기계 판독 수준 |
| 업종 분류 | 일반 / 금융 | XBRL 택소노미 및 재무제표 구조가 상이 |
| 정규화 계층 | L1(원본) / L2(매핑) / L3(통합) | 개념 정규화 깊이 |

## 핵심 영역 (6개)

1. **재무제표 구조** — 재무상태표, 손익계산서, 현금흐름표, 자본변동표, 포괄손익계산서의 구성 요소와 상호 관계
2. **계정과목 체계** — IFRS 택소노미 기반 계정과목의 계층 구조, 확장 계정(ENTITY_EXTENSION) 처리
3. **회계 항등식** — 자산=부채+자본, 당기순이익 연결, 현금흐름 검증 등 수학적 제약
4. **기간/시점 구분** — instant(시점)과 duration(기간) 속성의 구분 및 적용 규칙
5. **연결/별도 구분** — 연결재무제표와 별도재무제표의 범위, 연결 조정, 내부거래 제거
6. **주석 체계** — 주석(Note)의 역할, 본문 항목과의 참조 관계, 비정형 텍스트 처리

## 보조 영역 (4개)

1. **파생 지표** — ROE, 부채비율, EPS 등 기본 재무 항목에서 산출되는 분석 지표
2. **시계열 분석** — 동일 기업의 기간 간 비교, 추세 분석, 정정 공시 반영
3. **기업 간 비교** — 동종 업종 내 재무 항목 비교, 계정과목 매핑 차이 보정
4. **기업 지배구조** — 지분 구조, 관계기업, 종속기업 관계

## 규제/표준 (3개)

1. **IFRS 준수** — 국제재무보고기준 준수 여부 검증, 택소노미 버전 관리
2. **감사** — 감사 의견, 적정/한정/부적정/의견거절 분류
3. **공시 의무** — DART 기반 공시 체계, 공시 유형별 처리 규칙

## 필수 개념 범주 (6개)

| 범주 | 설명 | 예시 |
|---|---|---|
| 보고 주체 | 재무제표를 작성하는 법인 | 삼성전자, SK하이닉스 |
| 재무 항목 | 계정과목 단위의 금액 정보 | 매출액, 총자산, 당기순이익 |
| 보고 기간 | 재무 정보의 시간 범위 | 2024년 1분기, 2023년 연간 |
| 회계 정책 | 인식·측정·표시 방법 선택 | 재고자산 평가방법, 감가상각 방법 |
| 주석 정보 | 본문 항목의 세부 설명 | 우발부채 내역, 특수관계자 거래 |
| 외부 참조 | 택소노미·표준 연결 정보 | IFRS 택소노미 요소, FIBO 매핑 |

## 참조 표준 (5개)

1. IFRS Taxonomy (연도별 버전)
2. DART 전자공시 택소노미
3. XBRL 2.1 Specification
4. FIBO (Financial Industry Business Ontology)
5. K-IFRS 한국채택국제회계기준

## 편중 감지 기준 (6개)

1. 특정 재무제표(예: 재무상태표)에 질문의 70% 이상 집중 시 경고
2. 연결/별도 중 한쪽만 다룰 경우 경고
3. 시계열 질문 없이 단일 시점만 반복 시 경고
4. 주석 영역 완전 무시 시 경고
5. 파생 지표 없이 원시 항목만 다룰 경우 경고
6. 단일 업종(일반 또는 금융)만 다룰 경우 경고

## 학습 통합

### 원천 형식과 진실의 원천 (onto_dependency)
- XBRL이 1차 원천이나, 금융업종 등에서 XBRL 구조 불완전 시 HTML이 진실의 원천으로 전환됨
- 원천 형식 축(XBRL/HTML/PDF)은 단순 포맷 구분이 아니라 데이터 신뢰도 계층을 나타냄
- DART HTML이 유일한 원천인 경우 SPOF(단일 장애점) 위험 존재 — 대체 원천 확보 필요

### 업종별 택소노미 차이 (onto_structure, onto_pragmatics)
- 금융업 XBRL은 role 코드 체계가 일반 업종과 상이하여 동일 파서로 처리 불가
- 업종 분류 축(일반/금융)은 파싱 로직뿐 아니라 계정과목 매핑, 재무제표 구조 전체에 영향

### 정규화 계층 (philosopher)
- L1: 원본 택소노미 그대로 보존
- L2: 동의어 매핑, 확장 계정 정규화
- L3: 기업 간 비교 가능한 통합 개념 체계
- Concept 정규화가 관계(Relation) 정규화보다 선행되어야 함

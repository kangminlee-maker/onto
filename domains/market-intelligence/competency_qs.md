# Market Intelligence Domain — 체계 검증 질문

## 1. 목적

이 문서는 시장 정보 분석 체계가 올바르게 구성되었는지 검증하기 위한 질문 목록이다.
각 질문은 특정 설계 원칙 또는 학습 사항의 준수 여부를 확인한다.

## 2. 데이터 수집 검증

### Q1. source_tier 정합성
> 관련 학습: `onto_logic` — source_tier 허용 집합이 3곳에서 일치해야 함

- [ ] 수집 정책(collection_policy)에 선언된 source_tier 값 집합을 나열하라.
- [ ] 정책 게이트(policy_gate)에서 참조하는 source_tier 값 집합을 나열하라.
- [ ] 분석 프롬프트/지시문에서 사용하는 source_tier 값 집합을 나열하라.
- [ ] 위 세 집합이 **정확히 일치**하는가? 불일치가 있다면 어디에서 발생하는가?

### Q2. 데이터 소스 분류 완전성
- [ ] 모든 활성 데이터 소스에 source_tier가 부여되어 있는가?
- [ ] source_tier가 없는 데이터가 분석에 투입된 사례가 있는가?
- [ ] 데이터 소스 추가 시 source_tier 부여를 강제하는 메커니즘이 존재하는가?

### Q3. 수집 주기 유효성
- [ ] 각 데이터 소스의 수집 주기가 해당 데이터의 변동 주기에 적합한가?
- [ ] data_freshness 기준이 정의되어 있고, 만료된 데이터의 처리 정책이 있는가?

## 3. 분석 검증

### Q4. 분석 방법론 전제 조건
- [ ] 사용된 각 분석 방법(analysis_method)의 전제 조건이 명시되어 있는가?
- [ ] 전제 조건이 충족되지 않은 상태에서 분석이 수행된 사례가 있는가?

### Q5. 세그먼트 분류 기준
- [ ] 시장 세그먼트의 분류 기준이 명시적으로 선언되어 있는가?
- [ ] 분류 기준이 상호 배타적(MECE)인가?
- [ ] 분류 기준 변경 시 기존 분석 결과에 미치는 영향이 추적 가능한가?

## 4. 전략 도출 검증

### Q6. 정책 게이트의 선언-구현 연결
> 관련 학습: `onto_structure` — 정책 선언과 코드 검사 간 강제적 연결 필요

- [ ] policy_gate에 선언된 통과 조건이 무엇인가?
- [ ] 각 통과 조건에 대응하는 **실제 검증 로직**(코드, 체크리스트, 자동화)이 존재하는가?
- [ ] 선언만 존재하고 검증 로직이 없는 "선언-구현 갭"이 있는가?
- [ ] 정책 선언 변경 시 검증 로직의 동기화를 강제하는 메커니즘이 있는가?

### Q7. 전략 옵션 추적 가능성
- [ ] 각 strategic_option이 어떤 분석 결과에서 도출되었는지 추적할 수 있는가?
- [ ] 분석 데이터가 갱신되었을 때 영향받는 전략 옵션을 식별할 수 있는가?

## 5. 위험 평가 검증

### Q8. 위험 평가 독립성
> 관련 학습: `onto_coverage` — 위험 평가는 전략 설계의 부속이 아닌 독립 영역

- [ ] 위험 평가가 전략 도출과 **독립적으로** 수행될 수 있는가?
- [ ] 전략이 없는 상태에서도 위험 평가 결과가 유효한가?
- [ ] 위험 평가 결과가 전략 수정을 트리거할 수 있는 경로가 존재하는가?
- [ ] 위험 평가 담당자/프로세스가 전략 도출 담당자/프로세스와 분리되어 있는가?

### Q9. 위험 정량화 일관성
- [ ] risk_score 산출 공식(영향도 × 발생 가능성)이 모든 위험 요소에 동일하게 적용되는가?
- [ ] risk_threshold가 정의되어 있고, 초과 시 자동 알림/대응 트리거가 있는가?

## 6. 구조 전반 검증

### Q10. 분류 축 혼합 방지
> 관련 학습: `onto_semantics` — 노드 타입과 트레이트의 분류 축 혼합 방지

- [ ] 트레이트(source_tier, risk_level 등)가 노드 타입으로 사용된 사례가 있는가?

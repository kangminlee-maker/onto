# Visual Design Domain — 개념 사전 및 해석 규칙

## 구성 원리 핵심 용어

- Visual Hierarchy = 시각적 위계 → 크기, 색상, 대비, 위치 등 시각 수단을 사용하여 정보의 중요도 순서를 전달하는 체계. 사용자의 시선 이동 경로를 의도적으로 설계하는 것
- Gestalt Principles = 게슈탈트 원리 → 인간이 시각 요소를 집단으로 인지하는 방식에 관한 원리. 근접성(Proximity), 유사성(Similarity), 연속성(Continuity), 폐합(Closure), 전경-배경(Figure-Ground) 분리
- CRAP Principles = CRAP 원칙 → Contrast(대비), Repetition(반복), Alignment(정렬), Proximity(근접성). Robin Williams가 정리한 비디자이너를 위한 레이아웃 4원칙
- Negative Space = 네거티브 스페이스 (여백) → 디자인 요소가 배치되지 않은 빈 영역. 의도적으로 사용하면 시각적 호흡, 요소 간 관계 명확화, 시선 유도의 역할을 한다. "비어 있는 것"이 아니라 "비워 놓은 것"
- Golden Ratio = 황금비 (1:1.618) → 자연과 고전 미술에서 관찰되는 비율. 레이아웃, 타이포그래피 스케일, 이미지 구성에서 시각적 균형의 참조점으로 사용
- Rule of Thirds = 삼분법 → 화면을 가로·세로 3등분하여 교차점에 핵심 요소를 배치하는 구성 기법

## 타이포그래피 핵심 용어

- Typeface = 서체 → 동일한 디자인 원칙으로 설계된 글자 집합의 전체 패밀리. Helvetica, Noto Sans 등
- Font = 폰트 → 서체의 특정 굵기(weight)·스타일(style)·크기(size) 조합. Helvetica Bold 16px는 하나의 폰트
- Type Scale = 타이포그래피 스케일 → 본문 크기를 기준으로 비율(ratio)을 적용하여 생성하는 크기 단계 체계. 예: Minor Third(1.2), Major Third(1.25), Perfect Fourth(1.333)
- Line-height = 행간(줄 높이) → 텍스트 행 사이의 수직 간격. 본문은 1.4~1.6배, 제목은 1.1~1.3배가 일반적 가독성 기준
- Letter-spacing = 자간 → 글자 사이의 수평 간격. 커닝(kerning, 특정 글자 쌍)과 트래킹(tracking, 전체 텍스트)을 구분
- Readability = 가독성 → 텍스트 블록을 읽기 편안한 정도. 행간, 행장(줄 길이), 정렬 방식에 영향받음
- Legibility = 판독성 → 개별 글자를 구분하고 인식할 수 있는 정도. 서체 자체의 특성(x-height, counter, stroke contrast)에 영향받음
- Measure = 행장(줄 길이) → 한 줄의 텍스트가 차지하는 너비. 영문 기준 45~75자, 한글 기준 25~35자가 최적 가독성 범위

## 색채 핵심 용어

- Color Wheel = 색상환 → 색상의 관계를 원형으로 표현한 도구. 보색(complementary), 유사색(analogous), 삼원색 배합(triadic) 등의 조화 관계를 도출하는 기준
- Hue = 색상 → 색의 종류(빨강, 파랑 등). 색상환 위의 위치
- Saturation = 채도 → 색의 선명도. 높을수록 순색에 가깝고, 낮을수록 무채색에 가까움
- Lightness/Value = 명도 → 색의 밝기. 높을수록 흰색에, 낮을수록 검정에 가까움
- Contrast Ratio = 명도 대비 비율 → 전경색과 배경색의 상대적 밝기 차이. WCAG 기준 일반 텍스트 4.5:1 이상, 대형 텍스트 3:1 이상
- Semantic Color = 시맨틱 색상 → 기능적 의미가 부여된 색상. success(초록), error(빨강), warning(노랑), info(파랑) 등. 원시 색상값이 아닌 의미 기반 명명
- Color Token = 색상 토큰 → 색상값을 추상화한 변수. Primitive token(blue-500)과 Semantic token(color-error)의 2계층 구조
- 60-30-10 Rule = 색상 비율 규칙 → 주색 60%, 보조색 30%, 강조색 10%의 배분 원칙. 시각적 균형과 위계를 유지하는 경험 법칙

## 레이아웃 핵심 용어

- Grid System = 그리드 시스템 → 콘텐츠를 정렬하고 배치하기 위한 구조적 틀. 컬럼(column), 거터(gutter), 마진(margin)으로 구성
- Column Grid = 컬럼 그리드 → 화면을 수직 컬럼으로 분할하는 그리드. 12컬럼이 범용적(2, 3, 4, 6등분 가능)
- Modular Grid = 모듈러 그리드 → 수직 컬럼과 수평 행을 조합하여 모듈(module) 단위로 구성하는 그리드
- Gutter = 거터 → 컬럼 사이의 간격. 콘텐츠 블록 간 시각적 분리를 제공
- Breakpoint = 브레이크포인트 → 반응형 디자인에서 레이아웃이 변경되는 뷰포트 너비 기준점
- Responsive Design = 반응형 디자인 → 뷰포트 크기에 따라 레이아웃이 유동적으로 변형되는 설계 방식
- Adaptive Design = 적응형 디자인 → 미리 정의된 브레이크포인트에서 이산적으로 레이아웃이 전환되는 설계 방식

## 디자인 시스템 핵심 용어

- Design Token = 디자인 토큰 → 디자인 의사결정을 코드와 디자인 도구에서 공유 가능한 원자 단위로 추상화한 것. 색상, 크기, 간격, 그림자, 모션 등을 포함
- Primitive Token = 원시 토큰 → 값 자체를 이름으로 표현하는 최하위 토큰. 예: blue-500, spacing-16
- Semantic Token = 시맨틱 토큰 → 용도/의미를 이름으로 표현하는 토큰. 예: color-primary, spacing-component-gap. Primitive 토큰을 참조
- Component Token = 컴포넌트 토큰 → 특정 컴포넌트에 한정된 토큰. 예: button-bg-primary. Semantic 토큰을 참조
- Atomic Design = 아토믹 디자인 → Brad Frost의 5계층 컴포넌트 구성 방법론. Atom(버튼, 입력 필드) → Molecule(검색 바) → Organism(헤더) → Template(페이지 레이아웃) → Page(실제 콘텐츠)
- Variant = 변형 → 동일 컴포넌트의 시각적/기능적 변주. 크기(small/medium/large), 스타일(primary/secondary/ghost), 상태(default/hover/active/disabled)
- Design System Governance = 디자인 시스템 거버넌스 → 토큰/컴포넌트의 추가·변경·폐기 절차, 기여 규칙, 버전 관리 정책

## 접근성 핵심 용어

- WCAG = Web Content Accessibility Guidelines → W3C가 발행하는 웹 접근성 지침. Level A(최소), AA(표준), AAA(강화)의 3단계 적합성 수준
- Perceivable = 인식 가능 → WCAG 4원칙 중 첫 번째. 정보와 인터페이스 요소가 사용자에게 인식될 수 있어야 한다
- Color Blindness = 색각 이상 → 특정 색상을 구분하지 못하는 시각 특성. 적록 색약이 가장 흔함(남성 약 8%). 색상만으로 정보를 전달하면 안 됨
- Focus Indicator = 포커스 표시자 → 키보드 탐색 시 현재 선택된 요소를 시각적으로 표시하는 장치. 최소 2px 이상, 3:1 명도 대비
- Alt Text = 대체 텍스트 → 이미지의 내용을 텍스트로 기술하는 것. 스크린 리더 사용자에게 이미지 정보를 전달

## 브랜드 아이덴티티 핵심 용어

- Brand Identity = 브랜드 아이덴티티 → 브랜드가 의도적으로 구축하는 시각적·언어적 표현의 총체. 로고, 색상, 서체, 톤앤매너, 이미지 스타일 등
- Logo = 로고 → 브랜드를 대표하는 시각 기호. 워드마크(글자형), 심볼(도형), 콤비네이션(글자+도형)으로 구분
- Clear Space = 보호 공간 → 로고 주변에 다른 요소가 침범할 수 없는 최소 여백. 로고 자체의 비율로 정의 (예: 로고 높이의 1/2)
- Tone and Manner = 톤앤매너 → 브랜드의 시각적·언어적 어조와 분위기. "전문적이고 따뜻한", "대담하고 혁신적인" 등의 형용사 조합으로 정의
- Style Guide = 스타일 가이드 → 브랜드 시각 요소의 사용 규칙을 문서화한 것. 디자인 시스템보다 좁은 범위(주로 브랜드 시각 요소에 한정)

## 모션 핵심 용어

- Easing = 이징 → 애니메이션의 속도 변화 곡선. ease-in(느리게 시작), ease-out(느리게 끝남), ease-in-out(양쪽 느림). 자연스러운 움직임의 핵심
- Duration = 지속 시간 → 애니메이션이 실행되는 시간. 피드백(100~200ms), 전환(200~400ms), 복잡한 이동(400~700ms)이 일반적 범위
- Micro-interaction = 마이크로인터랙션 → 단일 작업에 대한 즉각적 시각 피드백. 버튼 클릭 반응, 토글 전환, 입력 확인 등
- Transition = 전이 → 한 상태에서 다른 상태로의 시각적 변화. 페이지 전환, 모달 열기/닫기, 레이아웃 변경 등

## 주의가 필요한 동형이의어

- "스타일": CSS 스타일(코드 구현) ≠ 시각 스타일(디자인 의도) ≠ 브랜드 스타일(톤앤매너)
- "컴포넌트": 디자인 컴포넌트(Figma 등 디자인 도구 내 재사용 요소) ≠ 코드 컴포넌트(React/Vue 등 구현 단위). 둘의 1:1 대응은 보장되지 않음
- "토큰": 디자인 토큰(시각 속성의 추상값) ≠ 인증 토큰(보안) ≠ 언어 토큰(NLP)
- "계층": 시각적 위계(정보 중요도) ≠ 컴포넌트 계층(Atomic Design 단계) ≠ 토큰 계층(primitive/semantic/component)
- "그리드": 레이아웃 그리드(페이지 구조) ≠ 아이콘 그리드(아이콘 설계 기준선) ≠ 데이터 그리드(표 형태 UI)
- "테마": 색상 테마(light/dark) ≠ 브랜드 테마(브랜드별 시각 변형) ≠ 시즌 테마(기간 한정 시각 변경)
- "일관성": 시각적 일관성(동일 의미에 동일 표현) ≠ 기능적 일관성(동일 패턴에 동일 동작) ≠ 플랫폼 일관성(OS 규범 준수)
- "대비": 명도 대비(접근성 기준) ≠ 시각적 대비(디자인 원칙으로서의 차이 강조)

## 해석 원칙

- "아름답다"는 디자인 품질 판단 기준이 아니다. 시각 디자인의 품질은 의도한 목적(정보 전달, 행동 유도, 브랜드 전달)의 달성 여부로 판단한다
- 접근성은 선택 사항이 아니라 기본 요건이다. 접근성 기준을 충족하지 않는 디자인은 완성된 디자인이 아니다
- 디자인 토큰의 3계층(primitive → semantic → component)에서 상위 계층은 하위 계층을 참조해야 한다. 시맨틱 토큰이 하드코딩된 값을 직접 포함하면 일관성 유지가 불가능해진다
- 서체 선택은 감각적 판단이 아니라 기능적 판단이다. 매체(화면/인쇄), 콘텐츠 유형(본문/제목/코드), 대상 사용자(연령/언어)에 의해 제약된다
- 디자인 도구(Figma 등)의 컴포넌트와 코드 컴포넌트는 동기화 대상이지 동일물이 아니다. 양쪽의 불일치를 허용할 범위와 동기화 주기가 정의되어야 한다
- 시각적 위계가 없는 디자인은 "모든 것이 중요하다"고 주장하는 것과 같다. 모든 것이 중요하면 아무것도 중요하지 않다
- 모션/애니메이션은 장식이 아니라 정보이다. 상태 변화를 전달하지 않는 모션은 인지 부하만 증가시킨다
- 색상만으로 의미를 전달하면 안 된다(WCAG 1.4.1). 형태, 텍스트, 아이콘 등 보조 수단이 반드시 동반되어야 한다

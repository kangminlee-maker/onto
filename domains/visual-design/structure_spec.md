# Visual Design Domain — 구조 명세

## 시각 디자인 체계 필수 요소

- **색채 체계**: 색상 팔레트, 주색/보조색/강조색, 시맨틱 색상, 명도 대비 기준
- **타이포그래피 체계**: 서체 선택, 타이포그래피 스케일, 행간/자간/행장 규칙
- **간격 체계**: 기본 단위(base unit), 간격 스케일, 여백 규칙
- **레이아웃 체계**: 그리드 시스템, 브레이크포인트, 반응형 규칙
- **아이콘 체계**: 스타일, 크기 체계, 의미 일관성
- **컴포넌트 체계**: 재사용 컴포넌트, 상태 정의, 변형(variant)
- **브랜드 시각 요소**: 로고 규칙, 브랜드 컬러, 브랜드 서체, 톤앤매너
- **접근성 기준**: WCAG 적합성 수준, 명도 대비, 포커스 표시자, 색상 외 전달 수단

## 필수 관계

- 모든 시맨틱 토큰은 최소 1개의 원시 토큰을 참조해야 한다 (토큰 참조 무결성)
- 모든 컴포넌트는 시맨틱 토큰을 통해 시각 속성을 받아야 한다 (토큰 기반 스타일링)
- 모든 인터랙티브 컴포넌트는 최소 5가지 상태(default/hover/active/focus/disabled)가 정의되어야 한다 (상태 완전성)
- 모든 텍스트-배경 색상 조합은 WCAG 명도 대비 기준을 충족해야 한다 (접근성 정합성)
- 시각 체계의 모든 요소는 브랜드 아이덴티티와 정합해야 한다 (브랜드 정합성)
- 모든 시각적 의미 전달에는 색상 외 보조 수단이 동반되어야 한다 (다중 감각 전달)

## 토큰 계층 구조

분류 축: **추상화 수준** — 값의 구체성에서 의미의 추상성으로 진행한다.

| 계층 | 명명 규칙 | 역할 | 예시 |
|---|---|---|---|
| Primitive | 색상명-단계 | 원시값 정의 | blue-500, gray-100, spacing-16 |
| Semantic | 용도-속성 | 의미/용도 부여 | color-primary, color-error, spacing-component-gap |
| Component | 컴포넌트-속성-상태 | 컴포넌트 한정 | button-bg-primary, input-border-focus |

- Primitive → Semantic → Component 방향으로만 참조한다 (단방향 의존)
- Component 토큰이 Primitive 토큰을 직접 참조하면 의미 계층 위반
- 테마 전환은 Primitive 계층에서 값을 교체하여 Semantic/Component 계층에 전파하는 구조

## 컴포넌트 계층 구조 (Atomic Design)

| 계층 | 정의 | 예시 |
|---|---|---|
| Atom | 더 이상 분해할 수 없는 최소 UI 요소 | 버튼, 입력 필드, 레이블, 아이콘 |
| Molecule | Atom의 의미 있는 조합 | 검색 바(입력 필드 + 버튼), 폼 필드(레이블 + 입력 필드 + 도움 텍스트) |
| Organism | Molecule과 Atom의 복합 구성 | 헤더(로고 + 내비게이션 + 검색 바), 카드 목록 |
| Template | Organism의 페이지 수준 배치 | 대시보드 레이아웃, 목록 페이지 레이아웃 |
| Page | Template에 실제 콘텐츠가 채워진 상태 | 실제 데이터가 적용된 대시보드 |

- 상위 계층은 하위 계층으로 구성된다. 상위 계층이 하위를 우회하여 원시 HTML/스타일을 직접 포함하면 재사용성 상실
- Atom 변경 시 해당 Atom을 포함하는 모든 상위 계층에 변경이 전파된다. 영향 범위 추적 필수

## 상태 체계

모든 인터랙티브 컴포넌트에 적용되는 상태 목록:

| 상태 | 시각적 표현 원칙 | 용도 |
|---|---|---|
| Default | 기본 외형 | 초기 상태 |
| Hover | Default 대비 미세한 변화 (배경 밝기, 그림자 등) | 마우스 오버 피드백 |
| Active/Pressed | Hover보다 강한 변화 (눌림 효과) | 클릭/탭 피드백 |
| Focus | 명확한 외곽선 (2px 이상, 3:1 대비) | 키보드 탐색 표시 |
| Disabled | 투명도 감소 또는 채도 저하 | 사용 불가 상태 |
| Loading | 진행 표시자 (스피너, 스켈레톤) | 비동기 작업 중 |
| Error | 시맨틱 에러 색상 + 보조 수단 | 유효성 검증 실패 |
| Selected | 강조 배경 또는 체크 표시 | 선택 확인 |

## 반응형 구조

| 구간 | 일반적 브레이크포인트 | 그리드 특성 |
|---|---|---|
| Mobile | ~767px | 4컬럼, 단일 컬럼 레이아웃 중심 |
| Tablet | 768px~1023px | 8컬럼, 2단 레이아웃 |
| Desktop | 1024px~1439px | 12컬럼, 다단 레이아웃 |
| Wide | 1440px~ | 12컬럼, 최대 너비 제한(max-width) |

- 콘텐츠가 브레이크포인트 전환 시 사라지면 안 된다. 재배치 또는 축소되어야 한다
- 타이포그래피 스케일이 뷰포트에 따라 조정되어야 한다 (유동적 타이포그래피 또는 구간별 스케일)
- 터치 대상 크기는 모바일에서 최소 44×44px를 충족해야 한다

## 도메인 경계 관리

- 시각 디자인 도메인은 "보이는 것"의 체계를 다룬다. UX 리서치(사용자 조사), 정보 설계(IA), 인터랙션 설계(동작 로직)는 별도 도메인
- 코드 구현 세부사항(CSS 속성, 프레임워크 API)은 소프트웨어 엔지니어링 도메인 소관. 본 도메인은 시각 명세를 제공하고, 구현은 참조 관계
- 콘텐츠 전략(텍스트 작성, 톤앤보이스)은 시각 디자인과 중첩되지만 별도 관심사. 본 도메인은 텍스트의 시각적 표현을 다루고, 내용은 다루지 않는다
- 사용성 평가(usability testing)의 결과는 시각 디자인 수정의 입력이 될 수 있으나, 평가 방법론 자체는 본 도메인 소관이 아니다

## 고립 노드 금지

- 정의된 색상 토큰 중 어떤 컴포넌트에서도 사용되지 않는 것 → 경고 (미사용 토큰)
- 정의된 컴포넌트 중 상태가 불완전한 것 → 경고 (상태 미정의 컴포넌트)
- 브랜드 가이드라인에 정의된 시각 요소 중 디자인 시스템에 반영되지 않은 것 → 경고 (브랜드 미반영)
- 디자인 토큰 중 시맨틱 계층 없이 컴포넌트에 직접 사용되는 원시 토큰 → 경고 (계층 우회)
- 접근성 기준이 적용되지 않은 색상 조합 → 경고 (대비 미검증)
- 디자인 도구에만 존재하고 코드에 구현되지 않은 컴포넌트 (또는 그 반대) → 경고 (동기화 미완료)

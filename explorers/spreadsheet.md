# Explorer 프로파일: 스프레드시트

## 탐색 도구
스프레드시트 파싱 도구 (MCP 서버 등)

## module_inventory 단위
시트 / 명명된 범위 / 테이블

## 구조 인식 범위
- 셀 값 (문자열, 숫자, 날짜)
- 수식 및 수식 간 참조 관계
- 서식 (배경색, 폰트, 테두리, 셀 병합)
- 시트 간 참조
- 데이터 유효성 검사 규칙
- 명명된 범위 (Named Range)
- 매크로/VBA 유무

## 소스 유형 판별 조건
- 파일 확장자: .xlsx, .xls, .csv, .ods
- MIME type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet

## 구조 인식 예시

올바른 보고:
> "A1:F1 셀이 병합되어 있고, 배경색 #4472C4, 폰트 Bold이며, 아래 행과 서식이 다르다"

> "B2 셀에 수식 =VLOOKUP(A2, Sheet2!A:C, 3, FALSE)이 있으며, Sheet2의 A~C 열을 참조한다"

해서는 안 되는 보고:
> "A1:F1은 테이블 헤더이다"

> "이 시트는 매출 보고서이다"

## detail 위치 표기
`"{설명} — {시트}:{셀범위}"`

예: `"수식 =SUM(B2:B10) — Sheet1:B11"`

## Phase 0.5 맥락 질문

- "색상이나 서식에 특별한 의미가 있나요? (예: 빨간 셀 = 미결, 회색 행 = 비활성)"
- "이 파일을 주로 누가 사용하나요?"
- "이미 정의된 도메인 용어집이 있나요?"
